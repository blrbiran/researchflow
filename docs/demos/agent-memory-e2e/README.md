# Agent Memory E2E Demo

This demo shows the minimum end-to-end ResearchFlow artifact chain for a sample topic:

> Agent memory systems for coding agents, with emphasis on safe experience reuse.

The demo is intentionally compact. Its purpose is to show what each phase hands to the next, not to act as a real literature review or submission-ready paper.

## Artifact chain

1. [01-literature-map.md](01-literature-map.md)
2. [02-structure-brief.md](02-structure-brief.md)
3. [03-draft-packet.md](03-draft-packet.md)
4. [04-review-packet.md](04-review-packet.md)
5. [05-submission-packet.md](05-submission-packet.md)

## How to use this demo

- Read the artifacts in order.
- Check that each artifact satisfies the required fields in `docs/workflow-contracts.md`.
- Use this folder as a template when creating future workflow demos or harness-level integration tests.
