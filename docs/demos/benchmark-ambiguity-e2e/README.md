# Benchmark Ambiguity E2E Demo

This demo shows the minimum end-to-end ResearchFlow artifact chain for a sample **benchmark / evaluation paper** topic:

> A benchmark for ambiguity handling in natural-language requests to text-to-visualization systems.

Unlike the agent-memory demo, this example is benchmark-centered rather than system-centered. Its purpose is to prove that the workflow contracts also fit evaluation-gap and benchmark-design papers.

## Artifact chain

1. [01-literature-map.md](01-literature-map.md)
2. [02-structure-brief.md](02-structure-brief.md)
3. [03-draft-packet.md](03-draft-packet.md)
4. [04-review-packet.md](04-review-packet.md)
5. [05-submission-packet.md](05-submission-packet.md)

## How to use this demo

- Read the artifacts in order.
- Compare its structure to the agent-memory demo.
- Notice that the same contract chain works even though the paper type is benchmark / evaluation rather than technical system design.
