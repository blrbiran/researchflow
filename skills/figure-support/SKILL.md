# Figure Support

Use this skill when the paper needs visual assets strong enough to support the argument.

## When to use

- The user wants to design a paper figure.
- The user is unsure what chart or diagram type fits a result.
- The user wants a figure audit before submission.
- The user wants help aligning figures with the section story.

## When not to use

- The paper structure is still unclear. Use `paper-structuring` first.
- The user only wants generic plotting advice outside a paper context.

## Figure classes

Prioritize the core figure types:
- motivated example / Figure 1
- solution overview / method or system diagram
- experimental results figures

## Workflow

1. Identify the figure's narrative job.
2. Choose the figure class and paradigm.
3. Sketch the layout, labels, and key emphasis.
4. Recommend the tool path.
5. Audit for submission-quality rules.

## Submission-quality rules

Check:
- vector-friendly output when possible
- readable font sizes after scaling
- color-blind-safe encoding
- self-contained captions
- honest axes and no chartjunk
- labels consistent with the paper's section language

## Output

Return:
- figure type and job
- recommended paradigm
- layout sketch
- labeling and caption guidance
- tool suggestion
- top figure-quality risks
