# Paper Review

Use this skill to critique a manuscript and produce a revision path.

## When to use

- The user asks for a paper review.
- The user wants revision priorities.
- The user wants a submission-readiness check.
- The user wants claims or citations sanity-checked.

## Review dimensions

1. Macro logic
2. Claim-evidence alignment
3. Literature coverage
4. Writing clarity and structure
5. Figures / tables / artifact readiness if present

## Workflow

1. Read the draft as a reviewer.
2. Identify critical blockers first.
3. Separate structural problems from language polish.
4. Build a revision list ordered by impact.
5. Recommend whether to revise structure, revise prose, or package artifacts next.

## Output

Return:
- critical issues
- major issues
- minor issues
- revision order
- recommended next phase
